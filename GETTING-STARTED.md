## 📋 完整使用流程

### 1. 首次使用（交互式模式）
```bash

# 2. 系统会询问：
```

## 🚨 常见问题和解决方案

### Q1: 安装后脚本不工作？
**解决方案:**
```bash
# 1. 检查依赖
./setup-v2.sh -c

# 2. 查看状态
./setup-v2.sh -s

# 3. 运行测试
./testing/test-approval-scripts.sh

# 4. 查看日志
tail -f /tmp/claude-approval.log
```

### Q2: 审批决策不符合预期？
**解决方案:**
```bash
# 1. 使用调试工具
./testing/debug-approval-script.sh -d 3 -v 脚本路径 测试输入

# 2. 检查项目类型识别是否正确
echo "项目类型: $(./setup-v2.sh -c 2\u003e\u00261 | grep "项目类型")"

# 3. 考虑更换环境类型
./setup-v2.sh -e 其他环境类型 -m 当前安装方式
```

### Q3: 多个项目需要不同配置？

```bash

./setup-v2.sh -e dev -m project -p ~/project-a
```

## 🎯 成功使用的关键

### 1. 理解项目类型
- Node.js项目需要支持npm/yarn命令
- Python项目需要支持pip/python命令
- Java项目需要支持mvn/gradle命令
- 通用项目使用基础安全策略

### 2. 选择合适的安装方式
- **个人使用**: 用户级安装（最灵活）

- **测试阶段**: CI/CD环境或智能审批
